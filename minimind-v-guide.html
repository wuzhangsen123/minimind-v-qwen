MiniMind-V：从 65M MiniMind 到 0.5B Qwen2.5 的完整迁移

Wu Zhangsen · 2026-05-23

1. 动机：为什么替换 LLM 骨架

原 MiniMind-V 的 LLM 骨架是 MiniMind 65M 参数模型。在纯文本测试中发现，该模型的语言能力非常有限：

  • 无法识别草的颜色、太阳的形状等基础常识
  • 回答经常偏移主题，生成质量不稳定
  • 65M 参数的容量天花板限制了 VLM 的视觉-语言对齐上限

相比之下，Qwen2.5-0.5B（494M 参数）经过大规模预训练，语言能力远超 MiniMind。将 LLM 从 65M 升级到 494M，预期可以显著提升 VLM 的识图准确率。

3. 三个关键设计决策

3.1 Projector 扩展：768→1536→896

原 projector 是单层 Linear(768→768)，改为 2 层 MLP 加 GELU 激活：

MMVisionProjector:
  LayerNorm(768)
  → Linear(768, 1536) → GELU
  → Linear(1536, 896)  # 896 = Qwen 的 hidden_size

扩展层让视觉特征有足够容量映射到 Qwen 的语义空间。

3.2 视觉嵌入融合：从 token 替换到 inputs_embeds

MiniMind 在 Transformer 层中手动替换 image token 的 hidden states。Qwen 方案直接在 embedding 层做替换：

text_embeds = llm.get_input_embeddings()(input_ids)
vision_embeds = siglip(pixel_values) → projector
text_embeds = splice(text_embeds, input_ids, vision_embeds)
outputs = llm(inputs_embeds=text_embeds, ...)

优点：Qwen LLM 本身不需要任何修改，标准 HuggingFace 接口。

3.3 Label 生成：基于 chat template 结构

MiniMind 用 token 模式匹配（找 <bos>assistant）来定位 assistant 回复位置。Qwen 的 template 不同，改为用 chat template 结构自动计算 prefix 长度：

prefix_convs = [c for c in conversations if c['role'] != 'assistant']
prefix_prompt = apply_chat_template(prefix_convs, add_generation_prompt=True)
prefix_len = len(tokenizer(prefix_prompt).input_ids)
labels = [-100] * prefix_len + input_ids[prefix_len:]

5. 6 个 Bug 记录

Bug #1: Label 对不齐 (loss=NaN, no grad_fn)

原因: MiniMind 的 <bos>assistant 模式匹配在 Qwen template 下失效，全部 labels = -100。

修复: 用 chat template 结构计算 prefix_len（_get_prefix_len 方法）。

Bug #2: Vision features 5D 维度

原因: Dataset collate 产生 [B, N, C, H, W]，SigLIP 期望 [B, C, H, W]。

修复: get_vision_features 中检测 dim==5，flatten 处理。

Bug #3: save_dir 路径解析

原因: init_qwen_vlm 的 save_dir='../out' 相对 CWD，未像 llm_path 那样用 proj_root 解析。

修复: 添加 if not isabs: save_dir = join(proj_root, save_dir.lstrip('./'))。

Bug #4: cannot pickle 'module' object

原因: qwen_vlm_checkpoint 遍历 **kwargs 时把 swanlab 模块写入 resume_data，无法序列化。

修复: 过滤 ModuleType 和 FunctionType。

Bug #5: _get_prefix_len 未替换 image 标记

原因: 旧代码先用 create_chat_prompt（替换了 <image>），随后被 apply_chat_template（未替换）覆盖。

修复: 在 apply_chat_template 前手动替换 prefix_convs 中的 <image>。训练 checkpoint 需重跑。

Bug #6: OOM Killer (WSL2 内存耗尽)

原因: num_workers=4 × 17GB/fork，加主进程 19GB，超过 WSL2 内存限制。

修复: num_workers 降到 2，checkpoint 完整，续训恢复。
